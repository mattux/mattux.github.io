Diciamo che più che altro sul telefono non ne sento l'esigenza, se mi serve un reminder mi creo un evento sul calendario e sono a posto. Però ad un certo punto ho pensato che vedere sul desktop del pc quanti giorni mi mancano per dare quel maledetto esame o quanti giorni ho a disposizione per inoltrare una certa domanda, mi può tornare utile.

E quindi ho sottratto tempo prezioso allo studio per fare in fretta e furia quello che forse è un accrocchio, ma che fa ciò che deve.


Il risultato


I miei prossimi impegni, comodamente sul mio desktop


Ingredienti e struttura

Per mostrare del testo a schermo ci serve il nostro fidato Conky che, tra le altre cose, sa leggere un file di testo... Allora sarà sufficiente creare un file di testo strutturato come sopra!

A noi piace automatizzare, dunque possiamo mettere le note in un file inserendo la data di "scadenza", dopodiché darlo in pasto ad uno script che ci crea il file di testo dei to-do da mostrare.

La struttura è la seguente:

Il file delle note

Questo file io l'ho pensato con una struttura molto semplice. Ogni riga rappresenta una nota ed è formattata così:

giorno/mese/anno/NOTA1
giorno/mese/anno/NOTA2
.
.
.
giorno/mese/anno/NOTAn

Lo script

Non fa altro che leggere riga per riga il file notes, creare la corrispondente riga per il file todos e accodarlo a quest'ultimo. Non aggiungo ulteriori dettagli e non copio qui il codice bash, spero che i commenti che ho inserito nello script siano sufficienti a chiarirne il funzionamento.
Potete trovarlo a questo link.
 
Se il file notes non è presente lo script si incarica di crearne uno da zero, ovviamente vuoto.
Crea anche un file todos che ci dice che non abbiamo impegni.

Per il momento non gestisce date di scadenza maggiori ad un anno dalla data corrente.

Non tiene conto delle note con una data precedente a quella corrente, dunque non è necessario "pulire" costantemente il file dai nostri vecchi impegni.


Come si usa

Basta scaricarlo, renderlo eseguibile, eseguirlo una prima volta e poi modificare il file notes inserendo i nostri to-do.
Lo script va eseguito periodicamente in modo da tenerlo sempre aggiornato.


Conky

Come si diceva sopra, lo scopo ultimo è mostrare il contenuto del file todos sul desktop mediante l'uso di Conky, anche perché un "robo" del genere ha poco senso se non posso vedere in comodità i miei to-do.
È sufficiente aggiungere queste righe nel nostro .conkyrc:

${texeci 900 sh /$HOME/notes_reader.sh}
${tail /$HOME/todos 30}

La prima riga dice che eseguiamo lo script ogni 900 secondi (ogni quarto d'ora) in un thread.
La seconda riga invece è semplicemente la lettura delle prime 30 righe (il massimo per Conky), del file todos.
